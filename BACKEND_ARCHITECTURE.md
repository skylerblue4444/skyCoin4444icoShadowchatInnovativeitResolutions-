# SKY4444 v10 Live - Backend Architecture Documentation\n\n## Overview\n\nThis document describes the advanced backend infrastructure and architectural patterns implemented in SKY4444 v10 Live. The backend is designed for high performance, scalability, and enterprise-grade security.\n\n---\n\n## Architecture Layers\n\n### 1. API Layer (tRPC)\n- **9 Backend Routers**: Dating, Marketplace, Livestream, Coin Economy, Admin, Payments, Feed, Messaging, Notifications\n- **100+ API Endpoints**: Fully typed with TypeScript\n- **Real-time Subscriptions**: WebSocket support for live updates\n- **Rate Limiting**: Per-endpoint rate limiting\n\n### 2. Real-time Communication Layer\n\n#### WebSocket Server (`server/websocket/ws-server.ts`)\n\n**Features**:\n- JWT-based authentication\n- Direct messaging between users\n- Typing indicators\n- Presence tracking (online/offline/away)\n- Channel subscriptions\n- Message history\n- Automatic reconnection handling\n- Heartbeat mechanism (30-second intervals)\n- Message queueing for offline users\n\n**Message Types**:\n```typescript\n- message: Direct user-to-user messages\n- typing: Typing indicators\n- presence: User status updates\n- subscribe: Channel subscriptions\n- unsubscribe: Channel unsubscriptions\n- ping/pong: Heartbeat\n```\n\n**Performance**:\n- Handles 1000+ concurrent connections\n- Sub-100ms message delivery\n- Automatic memory cleanup\n- Connection pooling\n\n#### WebSocket Client Hook (`client/src/hooks/useWebSocket.ts`)\n\n**Features**:\n- Automatic reconnection (5 attempts)\n- Exponential backoff\n- Message buffering\n- Connection state tracking\n- Type-safe message sending\n- Event handlers for all message types\n\n### 3. Background Job Processing Layer\n\n#### Job Processor (`server/jobs/job-processor.ts`)\n\n**Queue System**: Bull Queue with Redis\n\n**Job Types**:\n1. **Escrow Release**\n   - Automatic escrow fund release after order completion\n   - Wallet transaction creation\n   - Order status updates\n   - Retry logic with exponential backoff\n\n2. **Reward Distribution**\n   - Automatic reward claiming\n   - Wallet transaction creation\n   - Coin supply tracking\n   - Leaderboard updates\n\n3. **Payment Processing**\n   - Stripe payment processing\n   - Status tracking\n   - Webhook handling\n   - Failure recovery\n\n4. **Notification Delivery**\n   - Multi-channel delivery (email, push, SMS)\n   - User preference respect\n   - Quiet hours compliance\n   - Delivery tracking\n\n5. **Cleanup Tasks**\n   - Expired session cleanup\n   - Old log cleanup\n   - Temporary file cleanup\n   - Database optimization\n\n6. **Analytics**\n   - Daily statistics calculation\n   - User metrics aggregation\n   - Platform health metrics\n   - Performance tracking\n\n**Configuration**:\n- 10 concurrent workers per queue\n- 3 retry attempts with exponential backoff\n- Automatic dead-letter queue handling\n- Job history retention\n\n### 4. Caching Layer\n\n#### Cache Manager (`server/cache/cache-manager.ts`)\n\n**Features**:\n- Redis-backed distributed caching\n- Multiple data structures:\n  - Strings (key-value)\n  - Sets (unique members)\n  - Sorted Sets (leaderboards, rankings)\n  - Hashes (object storage)\n- TTL support (default 1 hour)\n- Cache-aside pattern\n- Pub/Sub messaging\n- Statistics tracking\n\n**Use Cases**:\n- User session caching\n- Leaderboard caching\n- Feed caching\n- Notification preferences caching\n- Rate limit counters\n- Real-time presence tracking\n\n**Performance**:\n- Sub-1ms cache hits\n- Automatic TTL expiration\n- Memory-efficient storage\n- Pub/Sub for real-time updates\n\n### 5. Security Layer\n\n#### Security Middleware (`server/middleware/security.ts`)\n\n**Rate Limiting**:\n```typescript\n- API: 100 requests per 15 minutes\n- Auth: 5 attempts per 15 minutes\n- Payment: 10 attempts per hour\n- Marketplace: 30 requests per minute\n- Messaging: 50 messages per minute\n```\n\n**Security Headers** (Helmet):\n- Content Security Policy (CSP)\n- HTTP Strict Transport Security (HSTS)\n- X-Frame-Options\n- X-Content-Type-Options\n- X-XSS-Protection\n\n**Input Validation**:\n- XSS prevention\n- SQL injection prevention\n- NoSQL injection prevention\n- Suspicious pattern detection\n\n**IP Filtering**:\n- Whitelist/blacklist support\n- Suspicious activity tracking\n- Automatic blacklisting after 10 violations\n\n**Audit Logging**:\n- All state-changing operations logged\n- User tracking\n- Timestamp recording\n- Action details capture\n\n**CSRF Protection**:\n- Token validation\n- Session-based tokens\n- Safe method exemption\n\n**Anomaly Detection**:\n- Rapid request detection\n- Large transfer detection\n- Failed login tracking\n- Suspicious pattern recognition\n\n---\n\n## Data Flow Patterns\n\n### 1. Real-time Messaging Flow\n```\nClient A (useWebSocket)\n  ↓\n  WebSocket Connection (JWT Auth)\n  ↓\n  Message Event\n  ↓\n  WebSocket Server (Message Handler)\n  ↓\n  Database Storage\n  ↓\n  Client B (WebSocket Connection)\n  ↓\n  Message Delivered\n```\n\n### 2. Background Job Flow\n```\nEvent Trigger (Order Complete, Reward Earned, etc.)\n  ↓\n  Add Job to Queue\n  ↓\n  Bull Queue (Redis)\n  ↓\n  Worker Processing\n  ↓\n  Database Updates\n  ↓\n  Notification Creation\n  ↓\n  User Notification\n```\n\n### 3. Caching Flow\n```\nAPI Request\n  ↓\n  Check Cache\n  ↓\n  Cache Hit? → Return Cached Data\n  ↓\n  Cache Miss? → Query Database\n  ↓\n  Store in Cache (with TTL)\n  ↓\n  Return Data\n```\n\n### 4. Security Flow\n```\nIncoming Request\n  ↓\n  Rate Limit Check\n  ↓\n  IP Filter Check\n  ↓\n  Input Validation\n  ↓\n  CSRF Check\n  ↓\n  Authentication\n  ↓\n  Authorization\n  ↓\n  Process Request\n  ↓\n  Audit Log\n```\n\n---\n\n## Database Schema\n\n### Core Tables\n- `users` - User accounts and profiles\n- `sessions` - User sessions\n- `auditLogs` - Audit trail\n\n### Dating Module\n-
